Base URL: /api/v1

Quy ước chung:

  • Định dạng: Tất cả request và response body đều sử dụng JSON.
  • Cấu trúc Response thành công:
    {
  "success": true,
  "message": "Thông báo thành công.",
  "data": { ... } or [ ... ]
}
  • Cấu trúc Response lỗi:
    {
  "success": false,
  "message": "Thông báo lỗi.",
  "errors": [ { "field": "ten_truong", "message": "chi_tiet_loi" } ]
}
  • Xác thực: Các endpoint yêu cầu xác thực phải được gửi kèm header: Authorization: Bearer <accessToken>.

Phần 1: Xác thực (Authentication)

Các endpoint này quản lý việc đăng ký, đăng nhập và các hoạt động liên quan đến tài khoản người dùng.

1.1. Đăng ký tài khoản mới

Tạo một tài khoản người dùng (customer) mới trong hệ thống.

  • Endpoint: POST /auth/register
  • Authentication: Public
  • Request Body:
    {
  "fullName": "Trần Văn An",
  "email": "an.tran@example.com",
  "password": "Password123!"
}
    FieldTypeDescriptionConstraints
    fullNamestringHọ và tên người dùng.Bắt buộc
    emailstringĐịa chỉ email.Bắt buộc, duy nhất
    passwordstringMật khẩu.Bắt buộc, tối thiểu 8 ký tự, có chữ hoa, chữ thường và số
  • Success Response (201 Created):
    {
  "success": true,
  "message": "Đăng ký tài khoản thành công.",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "fullName": "Trần Văn An",
      "email": "an.tran@example.com",
      "role": "customer"
    }
  }
}
  • Error Responses:
    • 400 Bad Request: Dữ liệu đầu vào không hợp lệ.
    • 409 Conflict: Email đã tồn tại trong hệ thống.

1.2. Đăng nhập

Xác thực thông tin người dùng và trả về một JSON Web Token (JWT) để sử dụng cho các yêu cầu sau.

  • Endpoint: POST /auth/login
  • Authentication: Public
  • Request Body:
    {
  "email": "an.tran@example.com",
  "password": "Password123!"
}
  • Success Response (200 OK):
    • Cấu trúc response tương tự như khi đăng ký thành công, chứa accessToken và thông tin user.
  • Error Responses:
    • 401 Unauthorized: Email hoặc mật khẩu không chính xác.
    • 429 Too Many Requests: Tài khoản bị tạm khóa do đăng nhập sai quá nhiều lần.

Phần 2: Người dùng (User Management)

Các endpoint để quản lý thông tin cá nhân của người dùng đã đăng nhập.

Lưu ý: Tất cả các endpoint trong mục này yêu cầu xác thực.

2.1. Lấy thông tin người dùng hiện tại

Lấy thông tin chi tiết của người dùng đang đăng nhập.

  • Endpoint: GET /user/me
  • Authentication: Required
  • Success Response (200 OK):
    {
  "success": true,
  "message": "Lấy thông tin người dùng thành công.",
  "data": {
    "id": 1,
    "fullName": "Trần Văn An",
    "email": "an.tran@example.com",
    "phoneNumber": "0987654321",
    "role": "customer",
    "status": "active",
    "createdAt": "2025-09-21T10:00:00Z"
  }
}

2.2. Cập nhật thông tin cá nhân

Cập nhật họ tên và số điện thoại của người dùng.

  • Endpoint: PUT /user/me
  • Authentication: Required
  • Request Body:
    {
  "fullName": "Trần Văn An Mới",
  "phoneNumber": "0123456789"
}
  • Success Response (200 OK):
    • Trả về đối tượng người dùng đã được cập nhật (tương tự 2.1).

2.3. Thay đổi mật khẩu

Thay đổi mật khẩu của người dùng đã đăng nhập.

  • Endpoint: PUT /user/change-password
  • Authentication: Required
  • Request Body:
    {
  "currentPassword": "Password123!",
  "newPassword": "NewPassword456!"
}
    FieldTypeDescriptionConstraints
    currentPasswordstringMật khẩu hiện tại.Bắt buộc
    newPasswordstringMật khẩu mới.Bắt buộc, tối thiểu 8 ký tự, có chữ hoa, chữ thường và số
  • Success Response (200 OK):
    {
  "success": true,
  "message": "Thay đổi mật khẩu thành công.",
  "data": null
}
  • Error Responses:
    • 400 Bad Request: Mật khẩu hiện tại không chính xác.
    • 422 Unprocessable Entity: Mật khẩu mới không đáp ứng yêu cầu bảo mật.

2.4. Lấy danh sách địa chỉ

Lấy tất cả địa chỉ giao hàng đã lưu của người dùng.

  • Endpoint: GET /user/addresses
  • Authentication: Required
  • Success Response (200 OK):
    {
  "success": true,
  "message": "Lấy danh sách địa chỉ thành công.",
  "data": [
    {
      "id": 1,
      "recipientName": "Trần Văn An",
      "phoneNumber": "0987654321",
      "streetAddress": "123 Đường ABC, Phường XYZ",
      "city": "Thành phố Hồ Chí Minh",
      "isDefault": true
    }
  ]
}

2.5. Thêm địa chỉ mới

Thêm một địa chỉ giao hàng mới cho người dùng.

  • Endpoint: POST /user/addresses
  • Authentication: Required
  • Request Body:
    {
  "recipientName": "Người nhận B",
  "phoneNumber": "0909090909",
  "streetAddress": "456 Đường DEF, Phường UVW",
  "city": "Thành phố Hà Nội",
  "isDefault": false
}
  • Success Response (201 Created):
    • Trả về đối tượng địa chỉ vừa được tạo.

2.6. Cập nhật địa chỉ

Cập nhật thông tin một địa chỉ đã có.

  • Endpoint: PUT /user/addresses/{addressId}
  • Authentication: Required
  • Path Parameters:
    ParameterTypeDescription
    addressIdnumberID của địa chỉ cần cập nhật.
  • Request Body:
    {
  "recipientName": "Người nhận C",
  "phoneNumber": "0911111111",
  "streetAddress": "789 Đường GHI, Phường KLM",
  "city": "Thành phố Đà Nẵng",
  "isDefault": true
}
  • Success Response (200 OK):
    • Trả về đối tượng địa chỉ đã được cập nhật.
  • Error Response:
    • 404 Not Found: Không tìm thấy địa chỉ với ID đã cho.

2.7. Xóa địa chỉ

Xóa một địa chỉ khỏi danh sách địa chỉ của người dùng.

  • Endpoint: DELETE /user/addresses/{addressId}
  • Authentication: Required
  • Path Parameters:
    ParameterTypeDescription
    addressIdnumberID của địa chỉ cần xóa.
  • Success Response (200 OK):
    {
  "success": true,
  "message": "Xóa địa chỉ thành công.",
  "data": null
}
  • Error Response:
    • 404 Not Found: Không tìm thấy địa chỉ với ID đã cho.
    • 400 Bad Request: Không thể xóa địa chỉ mặc định.

Phần 5: Giỏ hàng (Shopping Cart)

Các endpoint này cho phép người dùng quản lý giỏ hàng của họ. Giỏ hàng được liên kết với người dùng đã đăng nhập.

Lưu ý: Tất cả các endpoint trong mục này yêu cầu xác thực.

5.1. Lấy thông tin giỏ hàng

Lấy toàn bộ thông tin về giỏ hàng hiện tại của người dùng, bao gồm danh sách sản phẩm và tổng tiền.

  • Endpoint: GET /cart
  • Authentication: Required
  • Success Response (200 OK):
    {
  "success": true,
  "message": "Lấy thông tin giỏ hàng thành công.",
  "data": {
    "items": [
      {
        "productId": 101,
        "productName": "iPhone 15 Pro Max 256GB",
        "thumbnailUrl": "/images/iphone-15-pro-max.jpg",
        "price": "32000000.00",
        "quantity": 1,
        "lineTotal": "32000000.00"
      }
    ],
    "totalItems": 1,
    "totalPrice": "32000000.00"
  }
}

5.2. Thêm sản phẩm vào giỏ hàng

Thêm một sản phẩm mới vào giỏ hàng hoặc tăng số lượng nếu sản phẩm đã tồn tại.

  • Endpoint: POST /cart/items
  • Authentication: Required
  • Request Body:
    {
  "productId": 102,
  "quantity": 1
}
  • Success Response (200 OK):
    • Trả về toàn bộ thông tin giỏ hàng đã được cập nhật (tương tự 5.1).
  • Error Responses:
    • 400 Bad Request: Số lượng yêu cầu vượt quá số lượng tồn kho.
    • 404 Not Found: Không tìm thấy sản phẩm với productId đã cho.

5.3. Cập nhật số lượng sản phẩm trong giỏ

Thay đổi số lượng của một sản phẩm đã có trong giỏ hàng.

  • Endpoint: PUT /cart/items/{productId}
  • Authentication: Required
  • Path Parameters:
    ParameterTypeDescription
    productIdnumberID của sản phẩm cần cập nhật.
  • Request Body:
    {
  "quantity": 2
}
  • Success Response (200 OK):
    • Trả về toàn bộ thông tin giỏ hàng đã được cập nhật.

5.4. Xóa sản phẩm khỏi giỏ hàng

Xóa hoàn toàn một sản phẩm ra khỏi giỏ hàng.

  • Endpoint: DELETE /cart/items/{productId}
  • Authentication: Required
  • Path Parameters:
    ParameterTypeDescription
    productIdnumberID của sản phẩm cần xóa.
  • Success Response (200 OK):
    • Trả về toàn bộ thông tin giỏ hàng đã được cập nhật.

Phần 6: Đặt hàng (Checkout & Orders)

Các endpoint liên quan đến quy trình thanh toán và quản lý đơn hàng.

6.1. Tạo đơn hàng mới (Checkout)

Tạo một đơn hàng mới từ giỏ hàng hiện tại. Endpoint này hỗ trợ cả người dùng đã đăng nhập và khách vãng lai (guest checkout).

  • Endpoint: POST /checkout
  • Authentication: Optional (Nếu có token, đơn hàng sẽ được liên kết với user. Nếu không, sẽ là guest checkout).
  • Request Body:
    {
  "shippingInfo": {
    "recipientName": "Nguyễn Văn C",
    "phoneNumber": "0912345678",
    "email": "c.nguyen@example.com",
    "streetAddress": "789 Đường GHI, Phường KLM",
    "city": "Thành phố Đà Nẵng"
  },
  "voucherCode": "SALE50K",
  "paymentMethod": "COD"
}
    • Lưu ý: Nếu người dùng đã đăng nhập, có thể gửi addressId thay cho shippingInfo.
  • Success Response (201 Created):
    {
  "success": true,
  "message": "Đặt hàng thành công.",
  "data": {
    "orderId": 501,
    "orderCode": "UTEHUB-1695354800",
    "totalAmount": "31950000.00"
  }
}
  • Error Response:
    • 400 Bad Request: Giỏ hàng trống, thông tin giao hàng không hợp lệ, hoặc mã voucher không hợp lệ.

6.2. Lấy lịch sử đơn hàng (cho người dùng đã đăng nhập)

  • Endpoint: GET /orders
  • Authentication: Required
  • Query Parameters: Hỗ trợ phân trang (page, limit).
  • Success Response (200 OK):
    • Trả về một danh sách các đơn hàng với cấu trúc OrderSummaryResponse và thông tin phân trang.

6.3. Lấy chi tiết đơn hàng (cho người dùng đã đăng nhập)

  • Endpoint: GET /orders/{orderId}
  • Authentication: Required
  • Success Response (200 OK):
    • Trả về thông tin chi tiết của đơn hàng với cấu trúc OrderDetailResponse.

6.4. Tra cứu đơn hàng công khai

Cho phép bất kỳ ai (kể cả khách vãng lai) tra cứu thông tin đơn hàng bằng mã đơn hàng và email.

  • Endpoint: POST /orders/lookup
  • Authentication: Public
  • Request Body:
    {
  "orderCode": "UTEHUB-1695354800",
  "email": "c.nguyen@example.com"
}
  • Success Response (200 OK):
    • Trả về thông tin chi tiết của đơn hàng với cấu trúc OrderDetailResponse.
  • Error Response:
    • 404 Not Found: Không tìm thấy đơn hàng khớp với thông tin đã cung cấp.

Phần 7: Đánh giá (Reviews)

Các endpoint cho phép người dùng đã mua hàng có thể đánh giá sản phẩm.

7.1. Gửi đánh giá cho một sản phẩm

Tạo một bài đánh giá mới cho một sản phẩm. Chỉ những người dùng đã mua và đơn hàng đã hoàn thành mới có quyền thực hiện.

  • Endpoint: POST /products/{productId}/reviews
  • Authentication: Required
  • Path Parameters:
    ParameterTypeDescription
    productIdnumberID của sản phẩm cần đánh giá.
  • Request Body:
    {
  "rating": 5,
  "comment": "Sản phẩm rất tốt, giao hàng nhanh."
}
  • Success Response (201 Created):
    {
  "success": true,
  "message": "Gửi đánh giá thành công.",
  "data": {
    "id": 302,
    "rating": 5,
    "comment": "Sản phẩm rất tốt, giao hàng nhanh.",
    "user": { "fullName": "Trần Văn An" },
    "likeCount": 0,
    "createdAt": "2025-09-25T11:00:00Z"
  }
}
  • Error Responses:
    • 403 Forbidden: Người dùng chưa mua sản phẩm này hoặc đơn hàng chưa hoàn thành.
    • 409 Conflict: Người dùng đã đánh giá sản phẩm này trước đó.

7.2. Thích một bài đánh giá

Thích (Like) một bài đánh giá của người dùng khác.

  • Endpoint: POST /reviews/{reviewId}/like
  • Authentication: Required
  • Path Parameters:
    ParameterTypeDescription
    reviewIdnumberID của bài đánh giá cần thích.
  • Success Response (200 OK):
    {
  "success": true,
  "message": "Thích đánh giá thành công.",
  "data": {
    "likeCount": 11
  }
}
  • Error Response:
    • 409 Conflict: Người dùng đã thích bài đánh giá này.

7.3. Bỏ thích một bài đánh giá

Hủy lượt thích (Unlike) một bài đánh giá.

  • Endpoint: DELETE /reviews/{reviewId}/like
  • Authentication: Required
  • Path Parameters:
    ParameterTypeDescription
    reviewIdnumberID của bài đánh giá cần bỏ thích.
  • Success Response (200 OK):
    {
  "success": true,
  "message": "Bỏ thích đánh giá thành công.",
  "data": {
    "likeCount": 10
  }
}

Phần 8: Quản trị (Admin)

Các endpoint này chỉ dành cho người dùng có vai trò ADMIN.

Lưu ý: Tất cả các endpoint trong mục này đều có tiền tố /admin và yêu cầu xác thực với quyền Admin.

8.1. Quản lý Sản phẩm

  • POST /admin/products: Tạo sản phẩm mới.
    • Request Body: ProductRequest DTO.
    • Response: 201 Created với chi tiết sản phẩm vừa tạo.
  • PUT /admin/products/{productId}: Cập nhật thông tin sản phẩm.
    • Request Body: ProductRequest DTO.
    • Response: 200 OK với chi tiết sản phẩm đã cập nhật.
  • DELETE /admin/products/{productId}: Xóa mềm (soft-delete) sản phẩm.
    • Response: 204 No Content.

8.2. Quản lý Đơn hàng

  • GET /admin/orders: Lấy danh sách tất cả đơn hàng (có phân trang và lọc theo trạng thái).
    • Query Parameters: page, limit, status.
    • Response: 200 OK với danh sách đơn hàng và thông tin phân trang.
  • GET /admin/orders/{orderId}: Lấy chi tiết một đơn hàng.
    • Response: 200 OK với OrderDetailResponse.
  • PUT /admin/orders/{orderId}/status: Cập nhật trạng thái của một đơn hàng.
    • Request Body:
      {
  "status": "shipped"
}
    • Response: 200 OK với chi tiết đơn hàng đã cập nhật.

8.3. Quản lý Người dùng

  • GET /admin/users: Lấy danh sách tất cả người dùng (có phân trang).
    • Response: 200 OK với danh sách người dùng và thông tin phân trang.
  • PUT /admin/users/{userId}/status: Cập nhật trạng thái tài khoản người dùng (khóa/mở khóa).
    • Request Body:
      {
  "status": "locked"
}
    • Response: 200 OK với thông tin người dùng đã cập nhật.

8.4. Quản lý Mã giảm giá (Vouchers)

  • POST /admin/vouchers: Tạo một mã giảm giá mới.
    • Request Body: VoucherRequest DTO.
    • Response: 201 Created với chi tiết voucher vừa tạo.
  • PUT /admin/vouchers/{voucherId}: Cập nhật một mã giảm giá.
    • Request Body: VoucherRequest DTO.
    • Response: 200 OK với chi tiết voucher đã cập nhật.
  • DELETE /admin/vouchers/{voucherId}: Vô hiệu hóa một mã giảm giá.
    • Response: 204 No Content.

8.5. Dashboard & Thống kê

  • GET /admin/dashboard/summary: Lấy thống kê tổng quan cho dashboard.
    • Response (200 OK):
      {
  "success": true,
  "message": "Lấy thống kê dashboard thành công.",
  "data": {
    "totalRevenue": "125000000.00",
    "pendingOrders": 15,
    "totalProducts": 250,
    "newCustomersToday": 8
  }
}

8.6. Quản lý Danh mục (Categories)

  • POST /admin/categories: Tạo danh mục mới.
    • Request Body:
      {
  "name": "Điện thoại Android",
  "parentId": 1
}
    • Response: 201 Created với chi tiết danh mục vừa tạo.
  • PUT /admin/categories/{categoryId}: Cập nhật danh mục.
    • Request Body: Tương tự như tạo mới.
    • Response: 200 OK với chi tiết danh mục đã cập nhật.
  • DELETE /admin/categories/{categoryId}: Xóa mềm danh mục.
    • Response: 204 No Content.

8.7. Quản lý Thương hiệu (Brands)

  • POST /admin/brands: Tạo thương hiệu mới.
    • Request Body:
      {
  "name": "Xiaomi",
  "categoryId": 2
}
    • Response: 201 Created với chi tiết thương hiệu vừa tạo.
  • PUT /admin/brands/{brandId}: Cập nhật thương hiệu.
    • Request Body: Tương tự như tạo mới.
    • Response: 200 OK với chi tiết thương hiệu đã cập nhật.
  • DELETE /admin/brands/{brandId}: Xóa mềm thương hiệu.
    • Response: 204 No Content.
